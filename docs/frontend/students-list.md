# 学生列表模块前端实现说明

> 该文档为历史模块记录。`2026-07-04` 起，本模块已升级为 `人物档案`，当前实现以 `docs/frontend/people-profiles.md` 为准。

## 模块范围

本模块覆盖 MVP 的学生列表与学生资料新增 / 编辑。

包含：
- 学生卡片列表
- 新增学生
- 编辑学生
- 右侧抽屉表单
- 空状态
- 默认头像
- mock API 调用
- localStorage 测试数据同步

不包含：
- 搜索
- 筛选
- 删除
- 归档
- 头像上传
- 学生详情页
- 课程记录统计
- 真实后端持久化

## 页面入口

- 路由：`/students`
- AppShell 导航高亮：学生列表
- AppShell header 标题：学生列表

## 信息架构

AppShell header 已显示页面标题，因此内容区不重复大标题。

内容区顶部：
- 左侧说明：`管理可用于课程创建的学生档案。`
- 右侧主按钮：`新增学生`

内容区主体：
- 有学生：学生卡片网格
- 无学生：空状态

## 学生字段

### 必填字段

- 中文名
- 英文名
- 年龄
- 性别

### 非必填字段

- 兴趣爱好
- 学习目标
- 备注

### 不包含字段

- 英语水平

原因：
- 英语水平属于课程创建时的单次课程参数，不属于学生长期基础档案。

## 性别与头像

性别：
- 男
- 女

默认头像：
- 男 / 女各一张本地默认头像
- 由 AI 生成后放入本地 assets
- 本期不支持上传头像

## 学生卡片

采用卡片网格，不使用表格。

原因：
- 学生资料是“人”的档案，不是高密度流水数据。
- 卡片更适合展示头像、兴趣、学习目标和备注。
- 后续课程创建选择学生时，卡片形态也更容易复用。

### 卡片内容

每张卡片展示：
- 默认头像
- 中文名
- 英文名
- `年龄 · 性别`
- 兴趣爱好 tag
- 学习目标
- 备注
- 编辑 icon button

展示规则：
- 兴趣爱好按 tag 展示，最多 3 个，超出显示 `+N`
- 学习目标最多 2 行，超出省略
- 备注最多 2 行，超出省略

### 卡片交互

- 点击卡片整体：打开编辑抽屉
- 点击编辑 icon button：打开编辑抽屉
- hover：轻微边框高亮 / 浮起，动效 150-200ms

本期不做：
- 删除
- 归档
- 更多菜单

## 布局

### 顶部区域

顶部只放：
- 页面说明
- 新增学生按钮

不放搜索和筛选。

原因：
- 第一期简单实现，降低理解成本。
- 当前重点是让用户完成新增和编辑学生资料的闭环。

### 卡片网格

建议：
- 优先 3 列
- 宽度不足时自然变 2 列
- 卡片信息结构固定
- 避免不同卡片高度差过大

目标：
- 信息层级清晰
- 第一眼能识别学生
- 不把卡片做成密集表格

## 空状态

无学生时显示：

主文案：
`还没有学生`

辅助文案：
`新增学生后，即可在创建课程时选择。`

主按钮：
`新增学生`

## 新增 / 编辑抽屉

采用右侧抽屉，不使用新页面或居中弹窗。

原因：
- 字段数量适合抽屉。
- 新增 / 编辑后能直接回到列表。
- 新增和编辑可以复用同一套表单。

### 抽屉尺寸

- 宽度：`420px - 480px`
- 右侧进入
- 内容可滚动
- 底部固定操作区

### 抽屉标题

- 新增：`新增学生`
- 编辑：`编辑学生`

### 表单字段

- 中文名：单行输入，必填
- 英文名：单行输入，必填
- 年龄：数字输入，必填，不限制范围
- 性别：单选，男 / 女，必填
- 兴趣爱好：tag 输入，非必填
- 学习目标：多行文本，非必填
- 备注：多行文本，非必填

### 操作区

底部固定：
- 取消
- 保存

保存规则：
- 新增保存后：关闭抽屉，卡片出现在列表最前面
- 编辑保存后：关闭抽屉，原卡片更新
- 取消 / 关闭抽屉：如果表单有改动，弹确认：`放弃未保存的修改？`

## Mock API

前端调用：

- `GET /api/students`
- `POST /api/students`
- `PUT /api/students/:id`

当前使用 mock 数据。

为了刷新后不丢失验收数据，学生数据同步到 `localStorage`。

实现代码中必须保留注释：

`TODO: 前后端联调时删除 localStorage 测试逻辑，改为真实接口持久化。`

## 验收标准

实现完成后，需要满足：

- `/students` 显示学生列表页面
- 未登录访问 `/students` 会跳转 `/login`
- 页面无搜索和筛选
- 页面顶部显示说明和 `新增学生` 按钮
- 无学生时显示空状态和新增入口
- 有学生时显示卡片网格
- 学生卡片展示头像、中英文名、年龄、性别、兴趣、学习目标、备注
- 兴趣最多展示 3 个 tag，超出显示 `+N`
- 学习目标最多 2 行
- 备注最多 2 行
- 点击卡片或编辑按钮打开编辑抽屉
- 新增学生可保存并出现在列表最前面
- 编辑学生可保存并更新原卡片
- 表单改动后关闭抽屉时提示 `放弃未保存的修改？`
- 必填字段为空时不能保存
- 学生数据刷新后仍保留
- 代码中有 localStorage 测试逻辑删除提示注释
- 本期不出现删除、归档、搜索、筛选、头像上传入口
